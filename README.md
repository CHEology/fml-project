# ResuMatch

ResuMatch is a Streamlit app and ML pipeline for resume market assessment:
resume parsing, job retrieval, salary-range estimation, market clustering, and
actionable feedback.

The app can run with a small synthetic fallback catalog. Full retrieval,
salary, clustering, public-assessment, and wage features require local data and
model artifacts generated by the scripts below.

## Setup

```bash
uv sync
```

## Run

```bash
uv run streamlit run app/app.py
```

`app/app.py` is intentionally a thin bootstrap. Page rendering lives under
`app/pages/`, Streamlit demo flow helpers under `app/demo/`, runtime artifact
loading under `app/runtime/`, and shared resume assessment logic under
`ml/resume_assessment/`.

## Test And Quality

```bash
uv run ruff format --check app ml scripts tests
uv run ruff check app ml scripts tests
uv run pytest
```

The test suite is organized by source area:

```text
tests/app/       Streamlit UI, app runtime, chart components
tests/ml/        ML modules and resume assessment behavior
tests/scripts/   CLI/data-pipeline script behavior
tests/fixtures/  Small committed fixtures
```

A structural test enforces the repository rule that Python source files stay
under 1,000 lines.

## Data And Artifacts

Download Kaggle's LinkedIn Job Postings 2023-2024 dataset and place the
unzipped contents under `data/raw/`. See `data/README.md` for the expected raw
layout.

Build the processed job catalog:

```bash
uv run python scripts/preprocess_data.py
```

Expected outputs:

```text
data/processed/jobs.parquet
data/processed/salaries.npy
```

Build embeddings, FAISS retrieval metadata, and the retrieval index:

```bash
uv run python scripts/build_index.py
```

Expected outputs:

```text
models/job_embeddings.npy
models/jobs.index
models/jobs_meta.parquet
```

Build clustering artifacts:

```bash
uv run python scripts/build_clusters.py
```

Train optional model artifacts:

```bash
uv run python scripts/train_salary_model.py
uv run python scripts/train_resume_salary_model.py
uv run python scripts/train_quality_model.py
uv run python scripts/train_public_assessment_models.py
```

Optional public federal data:

```bash
uv run python scripts/load_onet_skills.py --download
uv run python scripts/load_bls_oews.py --download
```

If BLS blocks scripted downloads, download the national OEWS XLSX manually and
run:

```bash
uv run python scripts/load_bls_oews.py --input data/external/bls/national_M2024_dl.xlsx
```

Generated data, model checkpoints, embeddings, indexes, and eval artifacts are
gitignored. Do not commit raw Kaggle data, processed parquet files, model
weights, FAISS indexes, or downloaded federal data.

## Synthetic And Real Resume Evaluation

Generate synthetic resume/job pairs:

```bash
uv run python scripts/generate_synthetic_resumes.py \
    --jobs data/processed/jobs.parquet \
    --n 100 \
    --out data/eval/synthetic_resumes.parquet
```

The old CLI path is preserved, but the implementation now lives under
`scripts/synthetic_resumes/` with shared role taxonomy in `ml/taxonomy.py`.

Evaluate retrieval:

```bash
uv run python scripts/evaluate_retrieval.py
```

Normalize a public real-resume corpus:

```bash
uv run python scripts/load_real_resumes.py \
    --input data/raw/resumes/UpdatedResumeDataSet.csv \
    --out data/eval/real_resumes.parquet
```

Run the real-resume validation harness:

```bash
uv run python scripts/validate_on_real_resumes.py \
    --resumes data/eval/real_resumes.parquet \
    --index models/jobs.index \
    --meta models/jobs_meta.parquet \
    --onet-skills data/external/onet_skills.parquet \
    --bls-wages data/external/bls_wages.parquet \
    --salary-model models/resume_salary_model.pt \
    --salary-scaler models/resume_salary_model.scaler.json \
    --quality-model models/quality_model.pt
```

The validation harness degrades gracefully when optional artifacts are missing.
